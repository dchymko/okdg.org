Shared Security — August 24th 2010

Web Application Security by Adrian Schneider

Developing secure web applications requires the mindset that all user input is malicious, and it's up to the developer to properly sanitize anything that is to be used in the application. Adrian Schneider discusses common attack types, prevention, and also more complicated attacks and the theory behind them - all with examples.

View Presentation Slides

New Venue

We're happy to announce that we have lined up a venue for the August 24th event. This month's meetup (and hopefully all to come) will be in the board room at the Rotary Centre for the Arts. We have made arrangements with the catering company downstairs to have alcohol and a social area to be available from 9pm to 11pm for us. We don't mean to keep moving venues, but as we posted earlier our relations with our previous venue fell apart.

Please note the doors open at at 7pm (rather than 7:30pm) and presentations will start at 7:45pm sharp. Please do not show up too early before 7pm as we are setting up the room and dealing with facilities.

The Evening

The evening is starting off with Adrian Schneider discussing common security flaws in web applications with examples and solutions.

Following Adrian's presentation we're opening the floor to developers, designers, and companies to showcase their projects and what they've been working on. This is not a sales pitch, this is a time to show other people in the Okanagan what you're talented at. Maybe it's a site design you'd like critiqued? A web application you're trying to launch? An iPhone app? The floor is yours!

If you'd like to present work, please drop us a line and let us know! We can help with the computer / slides / projector / speakers for your presentation.

Looking forward to seeing you all there!
Shane, Derrick and Dustin
